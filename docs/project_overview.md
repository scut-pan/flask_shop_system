# Online Shopping Website

## 项目说明与技术方案（Project Overview & Technical Stack）

------

## 一、项目背景与目标

本项目为《网络应用架构设计与开发》课程实验项目，目标是**设计并实现一个可在线访问的在线购物网站**，完整覆盖以下核心能力：

- Web 应用架构设计
- 后端业务逻辑实现
- 数据库建模与数据持久化
- 用户认证与权限管理
- 基本部署与上线流程

项目需在限定时间内（约 5–10 天）完成，并满足课程给定的**基本功能要求**，同时保证代码结构清晰、可维护、可扩展。

------

## 二、总体技术选型说明（Technology Stack）

### 1. 后端语言与框架

**语言：** Python 3
 **Web 框架：** Flask

**选择 Flask 的原因：**

- 轻量、灵活，适合课程实验与中小型项目
- 支持服务端模板渲染，降低前端开发复杂度
- 学习曲线平缓，便于在开发过程中理解 Web 应用完整流程
- 与 Python 生态（数据库、邮件、部署）集成成熟

> 本项目不采用前后端分离架构，而是使用 Flask + 模板引擎的传统 Web 架构，以优先保证功能完整性和开发效率。

------

### 2. 数据库

**数据库类型：** MySQL
 **ORM 框架：** SQLAlchemy（Flask-SQLAlchemy）

**说明：**

- MySQL 满足课程对数据库的推荐要求
- ORM 方式有助于：
  - 提高开发效率
  - 降低 SQL 复杂度
  - 提高代码可维护性
- 数据库设计将作为实验报告中的重要组成部分

------

### 3. 前端技术

- HTML5
- CSS3
- Bootstrap 5（通过 CDN 引入）
- 少量 JavaScript（用于简单交互，如购物车）

**说明：**

- 前端以功能可用、结构清晰为目标
- 不追求复杂 UI 或前端工程化
- 所有页面由 Flask 使用模板引擎（Jinja2）渲染

------

### 4. 核心第三方库（预期使用）

- `Flask`：Web 应用框架
- `Flask-Login`：用户登录与会话管理
- `Flask-SQLAlchemy`：ORM
- `Werkzeug`：密码加密与安全相关工具
- `Flask-Mail`：订单确认与发货邮件发送
- （可选）`Flask-Migrate`：数据库迁移管理

------

## 三、系统架构设计思路

### 1. 架构模式

采用典型的 **MVC（Model-View-Controller）思想**：

- **Model（模型层）**
  - 定义数据库表结构
  - 封装数据访问逻辑
- **View（视图层）**
  - HTML 模板（Jinja2）
  - 展示数据与表单
- **Controller（控制层）**
  - Flask 路由函数
  - 处理请求、业务逻辑与响应

------

### 2. 功能模块划分（逻辑层面）

#### （1）用户模块

- 用户注册
- 用户登录
- 用户注销
- 登录态管理（Session）

#### （2）商品模块

- 商品列表展示
- 商品详情查看
- 商品搜索 / 查询（可选）

#### （3）购物车模块

- 添加商品至购物车
- 查看购物车
- 修改商品数量
- 从购物车删除商品

#### （4）订单模块

- 下单流程
- 订单状态管理
- 用户订单历史查询
- 管理员订单管理

#### （5）后台管理模块（销售管理）

- 商品的添加 / 删除 / 修改
- 订单管理
- 简单销售统计（如订单数量、总销售额）

------

## 四、数据库设计原则（概要）

- 每个核心业务对象对应一张数据表
- 表之间通过外键或逻辑关联
- 数据结构以“**够用、清晰、可解释**”为目标
- 数据库设计过程与结果需体现在实验报告中

**预期核心数据表包括：**

- 用户表（user）
- 商品表（product）
- 购物车表（cart）
- 订单表（order）
- 订单明细表（order_item）

------

## 五、部署与运行说明（目标）

### 1. 运行环境

- Linux 服务器（云服务器）
- Python 3.x
- MySQL
- Gunicorn + Nginx（生产环境）

### 2. 部署目标

- 网站可通过公网 IP 或域名访问

## 六、开发与学习原则（重要）

本项目遵循以下原则进行开发：

1. **以独立完成为目标**
   - 尽量自己编写代码
   - 不直接复制完整项目模板
2. **以理解为导向**
   - 每一个功能模块都清楚其作用与实现方式
3. **工具辅助而非替代**
   - 使用 Copilot / AI 工具：
     - 辅助理解
     - 提供语法示例
     - 帮助实现局部功能
   - 避免直接生成完整项目代码
4. **为实验报告服务**
   - 开发过程中的设计思路、关键实现方式需可解释、可复述
